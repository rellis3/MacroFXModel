🧠 COG V2 Engine — Persistent State Machine

Setup (Threshold1) → Risk (Gate2) → Trigger (NQ impulse + Gate3 direction), each persisting VALID/ARMED until explicitly invalidated — not snapshotted once at a fixed window. Live Monitor + Backtest Lab combined below.
V1 Backtest Lab → V1 Live Monitor → ← Dashboard
V2 Architecture: Real COG behavior is asynchronous — Setup can validate at 23:30 the previous night or 11:30am, entry is not a fixed candle, and if no entry happens ~5 minutes after the NY cash-equity open the setup is usually dead for the day. Setup Gate = computeThreshold1() evaluated every bar, continuously, no fixed window. Risk Gate = computeRiskGate(), frozen per calendar day (genuinely daily-resolution inputs) but tracked continuously from the start of the trading day. Trigger Gate = NEW — NQ's own price-action impulse (momentum burst, opening-range break, orderflow, volume) combined with Gate3's frozen daily direction, armed only inside the 14:20–14:35 UK window (NY open → NY open+5m). Entry fires when Setup ∩ Risk ∩ Trigger all agree on direction. All three gates' underlying math is reused unmodified from the V1 gate files — only the persistence/validity wrapper is new.
