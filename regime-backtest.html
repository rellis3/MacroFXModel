How the backtester works
① Load M1 candles from R2
Fetches raw M1 OHLC from R2 parquet files in 14-day chunks. No limit on date range — each pair holds years of history.
② Compute signals from candles
EMA20/50/200, Wilder ATR(70), vol z-score, 14-bar momentum → classify BULL/BEAR/RANGE/CHOP + conf/decay/vz/rl/score/bocpd/slope per bar. No external data needed.
③ Simulate bot logic
Replays V1 or V2 entry/exit gates against computed signals bar-by-bar. SL checked against each M1 bar's high/low. Config sliders directly control every gate threshold.
④ Walk through trades
Click any row in the Trade Log to zoom the chart to that trade (90 min context). Prev/Next in the nav bar above the analytics walk forward/back. Signal subcharts sync to show regime quality at entry.
Troubleshooting
Symptom Fix
No M1 candles Railway server not reachable, or R2 credentials missing. Check /api/vol-backtest/diagnose.
0 simulated trades Entry gates too tight for this pair/period. Try: lower Min Confidence (V1) or Entry Conf (V2), raise Max Vol-Z, or pick a longer date range. If the chart shows mostly RANGE/CHOP the bot wouldn't enter.
All RANGE/CHOP Classifier uses EMA20 vs EMA50 gap ÷ ATR. A very choppy period may not show directional regimes. Try a different date range or lower the EMA-gap threshold by lowering Min Confidence.
SL feels too wide/tight Adjust SL ATR Multiplier. Default 1.8 × ATR(70) M1 = same as 1.8 × ATR(14) M5 used by the live bot.
Config Reference
V1 — HMM Regime Bot
min_confidenceHMM conf % to allow entry (default 65)
candle_holdConsecutive polls regime must hold before entry (default 3)
vol_z_maxMax vol-Z at entry — blocks during vol spikes (default 2.5)
entry_decay_maxMax decay score at entry (default 0.25 = fresh regime)
decay_exitExit when decay hits this level (default 0.70)
sl_atr_multSL = ATR(70) × this multiplier (default 1.80)
range_exit_holdPolls in RANGE before exit (default 2)
window_start/endUTC trading window (default 07-20h)
V2 — Composite Score Bot
entry_confSession-adjusted confidence gate (default 70)
entry_score_minComposite score needed to enter (default 55)
bocpd_threshBlock entry if BOCPD change-point prob ≥ this (default 70%)
conf_floorExit X2: conf drops below this (default 45)
decay_exitExit X5: decay hits this threshold (default 0.70)
hold_score_minExit X11: score below this for N bars (default 40)
score_drop_threshExit X12: score drops ≥ this from entry score (default 20)
mfe_retraceExit X13: retrace 25% of peak when MFE ≥ 1R (default 0.25)
Decay score = conf_decay×0.40 + volz_decay×0.35 + rl_stall×0.25. Mixed-regime window → 0.90 immediately. V2 composite score weights: HMM 35% · BOCPD 20% · Session 15% · DXY 10% · Consensus 10% · ImplVol 5% · Credit 5%.
