⚡ Quick start
  1. Run a backtest on the 📐 Asia Range Backtest page — it saves results server-side automatically.
  2. Return here — the analysis page loads those results on page-open, no extra steps.
  3. Use the Global Filters to narrow the universe (e.g. just GBPUSD, tight confluence, London session).
  4. Read the three panels below to find your edge conditions and size TP/SL optimally.
Self-contained: all analysis is client-side JavaScript — no extra API calls. Every backtest re-run (on the backtest page) refreshes the data automatically when you reload this page.

Panel 1 — Conditional Pivot Heat Map

Pick any two dimensions for X and Y axes. Each cell shows the metric for trades matching both conditions.

  • X / Y axis: Fib Level, Session, Day of Week, Confluence, Zone, Range size, or Pair
  • Metric: Win Rate %, Expectancy, Profit Factor, or Sample Size (N)
  • Cells with <3 trades show — (statistically unreliable)
  • green = strong positive   red = negative
  • Intensity of colour scales with how far from the midpoint (50% for win rate, 0 for expectancy)

Tip: set X = Fib Level, Y = Session to see which fib bands work best in each session window.

Panel 2a — MFE: Optimal Take Profit

MFE (Max Favourable Excursion) = how far price moved in your favour before the trade closed, in R multiples.

  • Each bar = % of all filled trades that reached that R-level target
  • e.g. "80% reached 1R" means 80% of trades had enough room for a 1:1 TP
  • Suggested TP = the threshold with highest simulated expectancy under current filters

Panel 2b — MAE: SL Safety

MAE (Max Adverse Excursion) = how far price moved against you before resolving, in R multiples.

  • Each bar = % of trades that exceeded that drawdown level
  • Tightest safe SL = tightest SL where 80%+ of winners still survive
  • If winners regularly dip to 0.8R before recovering, a 0.5R SL will kill them — use MAE to calibrate

Panel 3 — Best Setup Finder

Enumerates every combination of selected dimensions, discards combos below min-trade threshold, and ranks the rest.

  • Group by: Ctrl/Cmd-click to select multiple dimensions — creates one row per unique combination
  • Rank by: Expectancy (recommended), Win Rate, Profit Factor, or Sharpe proxy
  • Min trades: raise to filter out low-sample noise (default 15)
  • ★ #1   ▲ #2   ◆ #3

Tip: start with Fib Level + Confluence + Session. Then drill into your best combo using Global Filters to see its MFE/MAE profile.

Key terms & metrics

  • R multiple: Entry-to-SL distance = 1R. TP at 2R = double your risk.
  • Expectancy: Average P&L per trade. Positive = edge. A 40% WR can still profit with high R:R.
  • Profit Factor: Gross wins ÷ gross losses. >1.5 good, >2 excellent.
  • Sharpe proxy: Mean return ÷ std dev — risk-adjusted quality score.
  • Tight confluence: Current and prior session fibs within tightPct% of pip threshold.
  • Regular confluence: Within normal pip threshold, but not tight.
  • Level zone: Deep neg (<−2), Outer neg (−2–0), Inner (0–1), Outer pos (1–2), Deep pos (>2).
  • Range buckets: Tiny <20 pip, Small 20–40, Medium 40–70, Large 70–120, Huge >120.
  • Monday aligned: Entry within 10 pips of Monday's high or low.

Global Filters strip

Every filter change instantly re-renders all three panels. Use this to ask conditional questions:

  • "What's the MFE profile for EURUSD tight-confluence London trades only?"
  • "Which fib levels work best on large-range Tuesdays?"
  • "Do Monday-aligned trades have better win rates in the inner zone?"

The stat chips (Filtered trades / Win rate / Expectancy / PF) update to show the overall edge for your current selection before you drill in further.

Filters only count filled trades — unfilled limit orders (price never reached entry) are excluded from all calculations.

Suggested workflow

  1. Run backtest over your full date range with all pairs and default settings.
  2. Open this page — check overall stats in Global Filters strip (no filters yet).
  3. Pivot: Fib Level × Session — spot which session/level combos are consistently green.
  4. Apply a Global Filter for one of those top combos (e.g. London + Inner zone).
  5. Read Panel 2 MFE to set TP; read MAE to validate SL is not too tight.
  6. Panel 3: group by Fib Level + Confluence + Session → note the ★ top setup.
  7. Go back to backtest page and tighten parameters around that setup — observe equity curve improvement.
