Range-Line Strategy

The Forecast-Level per-line confidence engine applied to Asia/Monday range levels — confluence modules stripped. Each line learns fade/follow/skip on after-cost expectancy (pooled IS → per-pair OOS), triple-barrier exit. Engine: js/rangeLineAnalyser.js.
Honest discipline: policy learned on in-sample only, applied out-of-sample; a cell trades only if its after-cost expectancy clears the margin. SKIP is a valid (and common) answer.

Book — out-of-sample (all pairs pooled policy)

The honest headline is the Portfolio Sharpe: daily portfolio returns ×√252, so all the same-day correlated touches across pairs/lines count as one day's risk — not thousands of independent bets (which is what a per-trade Sharpe ×√trades/yr falsely assumes). CAGR/Max DD shown at a fixed annual-vol target so they're leverage-honest. Win%/Expectancy/PF are per-trade descriptors (independence-free).

Live universe — survivors only

Rigor — walk-forward & stress (real vs artifact)

The headline Sharpe is breadth-inflated; THIS is how you judge real edge. IS→OOS degradation should hold (>~0.5). Cost stress should stay positive at 2–3× spread. Per-year should be broadly green. Deflated Sharpe discounts for how many settings were tried — >~95% means it clears its own search noise. A believable per-trade edge here looks like a low Sharpe that survives all four, not a high one that collapses.

E-ratio — is there a trend to ride? (exit study)

Per traded cell, average favourable vs adverse excursion to session close (not capped at the fixed barrier). E-ratio = MFE ÷ MAE. >~1.3 → price runs past the level, so a let-it-run / trailing exit would capture more than the current one-level TP (worth building). ≈1 → price pokes & reverts, the fixed TP is right. This answers whether the dynamic-TP / chandelier idea has a trend to ride before we build it.

Exit A/B — same entries, four exits (OOS + cost-stress)

Same learned policy; only the exit changes (follow trades only — fade keeps the fixed barrier). Held-position model — one trade through the trend (the HONEST Sharpe)

How you'd actually trade it: a follow signal opens one position per day/direction/source; it trails to exit; later same-direction touches while it's open are suppressed (no re-entry). This collapses the per-touch over-count, so trades/day falls and the Sharpe is finally tradeable — not breadth-inflated. Compare trades/day here vs the Exit-A/B card above. fixed-held = enter at first follow, one-level TP; structural/chandelier = trail. Judge on Sharpe @2–3× cost.

Bad-level scan — which (pair × level) reliably lose

The pooled policy learns ONE decision per level, so a level that's fine universally but bad on a specific pair still gets traded there. This scans every traded (pair × cell) and surfaces the reliably-negative ones. The veto is learned IN-SAMPLE only (drop pair-levels with negative IS expectancy on ≥30 trades) and applied OUT-OF-SAMPLE — so baseline→veto is an honest comparison. (Per-pair samples are small, so the veto can overfit — read the worst-list as diagnosis first.)
