🚀 Day one — bootstrap from Oanda history ⏱ 10–20 sec
Fetches ~13 days of H4 bars from Oanda (80 bars per pair), runs rolling beta estimation across those bars, classifies regime per window using EMA crossover, and derives targets immediately. Run this as soon as the bot is deployed.
# Preview first — see derived targets without pushing anything
python RegimeV2/beta_regime_table.py --backfill --dry-run

# Push regime table + update beta targets in KV
python RegimeV2/beta_regime_table.py --backfill --update-targets --url https://your-worker.workers.dev

# Double the history window (~26 days) for more regime diversity
python RegimeV2/beta_regime_table.py --backfill --bar-count 160 --update-targets --url https://your-worker.workers.dev

# Also save the synthetic records to beta_history.jsonl for future runs
python RegimeV2/beta_regime_table.py --backfill --save-backfill --update-targets --url https://your-worker.workers.dev
🔄 After weeks of live data — rebuild from real history ⏱ <2 sec
Once the bot has been running long enough to accumulate 30+ records per regime in bot/data/beta_history.jsonl, drop --backfill entirely. These targets will be more accurate because regime labels come from the actual HMM.
# Standard rebuild — derives targets from live history only
python RegimeV2/beta_regime_table.py --update-targets --url https://your-worker.workers.dev

# Require more samples per regime for higher-quality targets
python RegimeV2/beta_regime_table.py --update-targets --min-samples 50 --url https://your-worker.workers.dev
🔧 Other options
# Custom pairs (default covers EUR/GBP/USD/JPY/AUD/CAD/CHF/NZD)
python RegimeV2/beta_regime_table.py --backfill --pairs EURUSD GBPUSD USDCAD USDJPY --update-targets --url ...

# Practice/demo Oanda account
python RegimeV2/beta_regime_table.py --backfill --oanda-practice --update-targets --url ...

# OANDA_KEY can also be passed as env var instead of --oanda-key
OANDA_KEY=your_key python RegimeV2/beta_regime_table.py --backfill --update-targets --url ...

How to Run
The bot runs locally on any machine with MT5 installed and Python 3.10+.
python bot/position_hedge_bot.py --live --dashboard-url https://your-dashboard.com
Use --paper (default) to test without sending MT5 orders. State is saved in position_hedge_bot_state.json.
